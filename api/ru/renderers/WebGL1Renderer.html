[name]

Начиная с r118 [page:WebGLRenderer] автоматически использует контекст рендеринга WebGL 2. При обновлении существующего проекта до => r118, приложения могут перестать работать по двум причинам:

  • Код пользовательских шейдеров должен соответствовать стандарту GLSL 3.0.
  • Проверки расширения WebGL 1 должны быть изменены.
Если вы не можете выделить время на обновление кода, но все же хотите использовать последнюю версию, вы можете использовать [name]. Эта версия рендерера будет использовать контекст рендеринга WebGL 1.
