[page:Loader] →

[name]

Класс для загрузки [page:AnimationClip Анимационных клипов] в формате JSON. Для загрузки файлов используется внутренний [page:FileLoader].

Пример Кода

// instantiate a loader const loader = new v3d.AnimationLoader(); // load a resource loader.load( // resource URL 'animations/animation.js', // onLoad callback function(animations) { // animations is an array of AnimationClips }, // onProgress callback function(xhr) { console.log((xhr.loaded / xhr.total * 100) + '% loaded'); }, // onError callback function(err) { console.log('An error happened'); } );

Конструктор

[name]([param:LoadingManager manager])

[page:LoadingManager manager] — [page:LoadingManager loadingManager] для использования загрузчиком. По умолчанию [page:LoadingManager v3d.DefaultLoadingManager].

Создает новый [name].

Свойства

Общие свойства см. в базовом классе [page:Loader].

Методы

Общие методы см. в базовом классе [page:Loader].

[method:null load]([param:String url], [param:Function onLoad], [param:Function onProgress], [param:Function onError])

[page:String url] — путь или URL-адрес к файлу. Это также может быть [link:https://developer.mozilla.org/en-US/docs/Web/HTTP/Basics_of_HTTP/Data_URIs Data URI].
[page:Function onLoad] — Будет вызван после завершения загрузки. Аргументом будут загруженные [page:AnimationClip animation clips].
[page:Function onProgress] — Вызывается в процессе загрузки. Аргументом будет экземпляр XMLHttpRequest, который содержит байты .[page:Integer total] и .[page:Integer loaded].
[page:Function onError] — Будет вызван в случае ошибки при загрузке.

Начните загрузку из url и передайте загруженную анимацию в onLoad.

[method:Array parse]([param:JSON json])

[page:JSON json] — необходим

Парсит объект JSON и возвращает массив анимационных клипов. Отдельные клипы в объекте будут парситься с помощью [page:AnimationClip.parse].
