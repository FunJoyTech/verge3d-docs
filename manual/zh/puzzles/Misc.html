拼图参考

[anchor:open_web_page]

open web page(打开网页)

当此拼图被触发时，根据下拉选择，一个指定的URL会在一个新的或同一浏览器标签内打开。当从拼图编辑器中触发时，跳出当前页面前会弹出确认提示。

[anchor:social_share_link]

social share link(社交网站分享链接)

生成一个链接，以便在主流社交媒体上分享你的应用。

[anchor:call_JS_function]

call JS function(调用JS函数)

执行一个在应用的JavaScript代码中指定的函数。可以选择传递参数，作为函数参数使用。

为了在JavaScript代码中添加一个函数，用任何文本编辑器打开应用 .js 文件（例如， my_awesome_app.js 位于 verge3d/applications/my_awesome_app）。搜索 "prepareExternalInterface "并在该声明中添加您的函数（在大括号之间），这样它看起来像这样。

function prepareExternalInterface(app) { app.ExternalInterface.myJSFunction = function(numericArg, textArg) { alert('Got some params from Puzzles: ' + numericArg + ' and ' + textArg); } } [anchor:when_called_from_JS]

when called from JS(当从JS调用时)

已弃用。使用*v3d.puzzles.procedures*对象来直接调用拼图编辑器中定义的过程。 function runCode(app) { v3d.puzzles.procedures['myProcedure'](); }

[anchor:load_data]

load data(载入数据)

试图从一个指定的位置加载数据。不管尝试是否成功，"一旦准备好做 "槽中的拼图都会被解释。检索到的数据可以通过 加载数据 拼图进行访问。

[anchor:send_data]

send data(发送数据)

试图使用异步的 POST HTTP 请求将指定的数据发送到一个远程位置。不管尝试是否成功，"一旦准备好做 "槽中的拼图都会被解释。如果有任何响应数据，可以通过 loaded data 拼图访问。

[anchor:loaded_data]

loaded data(已加载的数据)

返回由 加载数据 或 发送数据 拼图获取的数据。

[anchor:read_JSON]

read JSON(读取JSON)

将文本（或 数据URI）解释为 JavaScript Object Notation(JavaScript对象符号) 数据，并以字典形式返回。

[anchor:read_CSV]

read CSV(读取JSON)

将文本（或 数据URI）解释为 comma-separated values(逗号分隔的值)。返回一个表格行的列表，每行表示为一个值的列表。分隔符可以通过下拉选择，以对应CSV文件的导出设置。"从行 "值表示从顶部开始跳过多少行。

表的行和值是通过它们的数字索引从0开始访问的。

[anchor:save_state]

save state(保存状态)

保存指定对象的状态和/或由其名称指定的变量值。对象被克隆并存储在内存中。变量的值被检索并存储在内存中，每个指定的变量名称都是如此。

如果此拼图被多次调用，那么这些状态将被依次保存，这样就可以通过 undo state(撤销状态) 拼图返回到之前的任一状态。

[anchor:undo_state]

undo state(撤销状态)

恢复用 save state(保存状态) 拼图保存的对象和/或变量的状态。

如果这个拼图被多次调用，状态会从保存的序列中恢复（如果有的话），这使得我们可以从堆栈中返回到任何先前保存的状态。

[anchor:all_variable_names]

all variable names(所有变量名称)

返回一个包含拼图中使用的所有变量名称的列表。

[anchor:variable_value_by_name]

variable value by name(对应名称的变量的值)

返回指定名称的变量的值。此拼图的工作方式与标准的变量值拼图类似，但不需要从预定义的下拉菜单中选择一个变量。
