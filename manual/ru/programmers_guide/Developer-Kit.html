Комплект Разработчика Verge3D

Комплект разработчика Verge3D (DevKit) - это набор примеров, инструментов и исходного кода, которые поставляются с пакетом Verge3D Enterprise. Этот набор поможет вам выполнять различные задачи разработчика, такие как:

  • Изучение и обзор внутренних компонентов набора инструментов Verge3D.
  • Модификация Verge3D и использование модифицированных версий в собственных 3D-приложениях.
  • Расширение API Verge3D с помощью собственных классов и методов.
  • Разработка новых блоков Пазлов.
  • Совершенствование экспортера Verge3D glTF, добавляя свои собственные данные или параметры.
  • Разработка своего собственного конфигуратора или системы e-commerce для WordPress используя плагин Verge3D for WordPress в качестве референса.

Комплект разработчикаDeveloper предназначен для опытных программистов, желающих модифицировать движок. Он не нужен для создания приложений Verge3D со стандартным набором функций.

Установка и Настройка

Возьмите архив DevKit из области загрузки панели инструментов вашей учетной записи. Распакуйте этот архив в выбранную вами директорию.

Если вы собираетесь собрать движок Verge3D или Пазлы из исходного кода, вам понадобится утилита *npm*, которая является частью среды разработки *Node.js*. Вы можете установить ее с помощью этих инструкций.

Наконец, перейдите в распакованный каталог DevKit и выполните следующую команду для установки зависимостей:

npm install [anchor:Structure]

Структура DevKit

DevKit включает следующие каталоги:

Каталог Описание
addons/verge3d Исходный код дополнения для Blender (Verge3D только для Blender).
build Готовые скрипты движка Verge3D и дополнительные модули (ammo.js и т.д.).
max_plugin Исходный код плагина для 3ds Max (Verge3D только для 3ds Max).
maya_plugin Исходный код плагина для Maya (только Verge3D для Maya).
puzzles Готовые скрипты и исходный код редактора Puzzles.
python Python 3 и служебные модули.
src Исходный код движка Verge3D.
templates Шаблоны приложений Verge3D.
wordpress Исходный код плагина Verge3D for WordPress.
xz Исходный код модуля сжатия XZ.

Сборка Verge3D

Перед компиляцией Verge3D, пожалуйста, убедитесь что вы установили *npm* и все необходимые зависимости, как описано выше.

Чтобы скомпилировать неоптимизированный билд Verge3D, что хорошо подходит для целей тестирования и отладки, выполните следующую команду в директории DevKit:

npm run build

Вы можете найти скомпилированные модули в директории *build* вашего DevKit. Чтобы протестировать его, просто скопируйте модуль движка *v3d.js* в ваше приложение внутри Verge3D, например:

cp build/v3d.js ~/verge3d/applications/my_awesome_application/

Чтобы скомпилировать оптимизированную версию Verge3D (она же релизная версия), выполните следующую команду в директории DevKit:

npm run build-release

Для сборки логического редактора Пазлов используются следующие команды. Для создания неоптимизированной версии введите:

npm run build-puzzles

Чтобы собрать оптимизированную версию, выполните команду:

npm run build-puzzles-release

Скомпилированный скрипт редактора Пазлов находится в директории *puzzles* вашего DevKit. Чтобы применить его в Verge3D, скопируйте все содержимое этого каталога в каталог *puzzles* в вашей директории Verge3D (заменяя уже существующие там файлы).

Добавление Нового Метода в JavaScript API

Допустим, вы хотите добавить еще один метод в класс *App*. Например, метод *printHelloWorld()* должен вывести что-то на консоль браузера:

App.printHelloWorld()

Класс *App* реализован в модуле, расположенном по адресу *src/extras/App.js*. Откройте его и добавьте следующий код в конец файла, прямо перед закрывающими скобками класса *App*. Не забудьте добавить еще одну запятую там, где заканчивается предыдущий метод.

printHelloWorld: function() { console.log('Hello World!'); }

Соберите и скопируйте среду выполнения Пазлов в папку с установленным Verge3D. Теперь вы можете попробовать новый добавленный блок в редакторе Пазлов.

[anchor:Custom_API_Class]

Добавление Нового Класса в JavaScript API

Чтобы дополнить API Verge3D каким-либо пользовательским классом, создайте .js файл с именем вашего класса в соответствующем подкаталоге папки *src* DevKit (например, src/extras/MyAwesomeClass.js). В этом .js-файле реализуйте свой класс - вы можете вдохновиться уже существующими классами. Наконец, зарегистрируйте свой класс в глобальном пространстве имен *v3d*, добавив следующую строку в файл *src/v3d.js*:

export { MyAwesomeClass } from './extras/MyAwesomeClass.js';

Соберите и скопируйте среду выполнения Verge3D в ваше приложение и попробуйте ее в консоли браузера.

var myAwesomeStuff = new v3d.MyAwesomeClass(); myAwesomeStuff.whatEver(); [anchor:Puzzles_Block]

Добавление Новых Пазлов

Чтобы создать пользовательский блок Пазлов, отредактируйте файл *puzzles/src/puzzles_blocks.js*. Вы можете вдохновиться уже существующими пазлами. Зарегистрируйте его в соответствующей категории панели инструментов в верхней части файла.

Соберите и скопируйте среду выполнения Пазлов в папку с установленным Verge3D. Теперь вы должны найти свой новый пазл в панели инструментов Редактора Пазлов.
